# Stack (스택)

자료구조 stack에 대해서 학습하는 공간입니다

## 개요

스택(Stack)은 데이터를 순서대로 쌓아 올린 형태의 자료구조입니다. 가장 나중에 들어온 데이터가 가장 먼저 나가는 LIFO(Last In First Out) 구조를 따릅니다.

## 주요 특징

- LIFO(Last In First Out): 후입선출 방식
- 한쪽 끝(top)에서만 데이터의 삽입과 삭제가 발생
- 실생활 예시: 접시 쌓기, 브라우저 뒤로가기 기능

## 주요 연산

### push
- 스택의 맨 위(top)에 새로운 요소를 추가
- 시간 복잡도: O(1)

### pop
- 스택의 맨 위(top)에 있는 요소를 제거하고 반환
- 시간 복잡도: O(1)

### peek (또는 top)
- 스택의 맨 위(top)에 있는 요소를 제거하지 않고 조회
- 시간 복잡도: O(1)

### isEmpty
- 스택이 비어있는지 확인
- 시간 복잡도: O(1)

## 구현 방법

스택은 다음과 같은 방법으로 구현할 수 있습니다:

1. 배열(Array) 기반 구현
2. 연결 리스트(Linked List) 기반 구현

## 활용 사례

실제 애플리케이션 개발에서 스택을 사용할 수 있는 구체적인 사례들입니다.

### 텍스트 에디터 - 실행 취소/다시 실행 (Undo/Redo)
**앱 예시**: 노션, MS Word, 구글 문서, VS Code
- **기능**: 사용자가 입력한 작업을 취소하거나 다시 실행
- **구현**:
  - undoStack: 사용자의 모든 작업(타이핑, 삭제 등)을 push
  - Ctrl+Z 누르면 undoStack에서 pop하여 작업 취소
  - redoStack에 취소된 작업을 push하여 다시 실행 가능
- **사용 시점**: 텍스트 편집기, 그림판, 디자인 툴 등 사용자 작업 이력 관리가 필요할 때

### 웹 브라우저 - 페이지 네비게이션
**앱 예시**: Chrome, Safari, Firefox
- **기능**: 뒤로가기/앞으로가기 버튼
- **구현**:
  - historyStack: 방문한 URL을 차례로 push
  - 뒤로가기 버튼 클릭 시 현재 페이지를 pop하고 이전 페이지로 이동
  - 새 페이지 방문 시 현재 페이지를 push하고 새 페이지로 이동
- **사용 시점**: 네비게이션 히스토리를 관리하는 모든 앱 (파일 탐색기, 갤러리 앱 등)

### 코드 에디터 - 괄호 매칭 및 자동완성
**앱 예시**: VS Code, IntelliJ, Sublime Text
- **기능**: 괄호 짝 맞추기, 자동 닫기, 문법 오류 검사
- **구현**:
  - 여는 괄호 `(`, `{`, `[` 를 만나면 스택에 push
  - 닫는 괄호 `)`, `}`, `]` 를 만나면 스택에서 pop하여 짝이 맞는지 확인
  - 스택이 비어있지 않으면 괄호가 닫히지 않은 것으로 오류 표시
- **사용 시점**: JSON 파서, HTML/XML 파서, 코드 lint 도구 개발 시

### 계산기 앱 - 수식 계산
**앱 예시**: iOS 계산기, 구글 계산기, 공학용 계산기
- **기능**: 복잡한 수식을 올바른 순서로 계산 (후위 표기법 변환)
- **구현**:
  - 중위 표기식(3 + 4 * 2)을 후위 표기식(3 4 2 * +)로 변환
  - 연산자 우선순위에 따라 스택 사용
  - 후위 표기식을 스택으로 계산
- **사용 시점**: 수식 파서, 수학 계산 기능이 필요한 앱

### 모바일 앱 - 화면 스택 관리
**앱 예시**: 인스타그램, 배달의민족, 카카오톡
- **기능**: 화면 전환 및 뒤로가기 버튼 동작
- **구현**:
  - 새 화면 열 때마다 현재 화면을 스택에 push
  - 뒤로가기 버튼 누르면 스택에서 pop하여 이전 화면으로
  - Android의 Activity Stack, iOS의 Navigation Stack
- **사용 시점**: 멀티 페이지 앱, 네비게이션 플로우가 있는 모든 모바일 앱

### 게임 - 상태 저장 및 복구
**앱 예시**: 체스 게임, 퍼즐 게임, 전략 게임
- **기능**: 한 수 되돌리기, 게임 상태 복구
- **구현**:
  - 각 턴마다 게임 상태를 스택에 push
  - 되돌리기 버튼 클릭 시 스택에서 pop하여 이전 상태로 복원
  - 세이브 포인트 시스템 구현
- **사용 시점**: 턴제 게임, 퍼즐 게임, 체스/장기 등

### 이메일/메시지 앱 - 임시 저장
**앱 예시**: Gmail, Outlook, 카카오톡
- **기능**: 작성 중인 메시지 임시 저장 및 복구
- **구현**:
  - 메시지 작성 시작 시 임시 저장 스택에 push
  - 뒤로가기 시 "임시 저장하시겠습니까?" 팝업 후 스택에 저장
  - 다시 작성 화면 진입 시 스택에서 pop하여 복구
- **사용 시점**: 양식 입력, 댓글 작성, 긴 텍스트 입력이 필요한 앱

### 파일 시스템 - 경로 탐색
**앱 예시**: 파일 탐색기, FTP 클라이언트
- **기능**: 폴더 이동 및 상위 폴더로 이동
- **구현**:
  - 폴더 진입 시 현재 경로를 스택에 push
  - 상위 폴더 버튼 클릭 시 스택에서 pop
  - 경로 breadcrumb 표시
- **사용 시점**: 파일 관리자, 드라이브 앱, 계층 구조 네비게이션

### 알고리즘 - 깊이 우선 탐색 (DFS)
**앱 예시**: 미로 찾기 게임, 그래프 분석 도구
- **기능**: 그래프나 트리를 깊이 우선으로 탐색
- **구현**:
  - 시작 노드를 스택에 push
  - 스택이 빌 때까지 pop하며 인접 노드들을 push
  - 방문 여부를 체크하며 탐색
- **사용 시점**: 경로 찾기, 네트워크 분석, 미로 탐색 등
